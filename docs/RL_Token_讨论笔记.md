# RL Token 论文讨论笔记

> 论文：RL Token: Bootstrapping Online RL with Vision-Language-Action Models
> 作者：Charles Xu et al., Physical Intelligence

---

## Q1: RLT 的核心方法是什么？

RLT 的核心思想是：**从冻结的大模型 VLA 中提取紧凑表示（RL token），在其上训练轻量级 actor-critic 进行在线 RL，将 RL 转化为对 VLA 动作的"局部精化"而非无约束探索。**

### 第一阶段：适配 VLA，暴露 RL Token

在预训练 VLA 的最终层嵌入之上，附加一个**编码器-解码器 Transformer**：

- **编码器**：接收 VLA 的所有 token 嵌入 $\mathbf{z}_{1:M}$ 加上一个可学习的特殊 token $\mathbf{e}_{\mathrm{rl}}$，输出该特殊 token 位置的嵌入 $\mathbf{z}_{\mathrm{rl}}$ 作为 RL token
- **解码器**：自回归地从 $\mathbf{z}_{\mathrm{rl}}$ 重建原始 VLA 嵌入，形成信息瓶颈——RL token 必须压缩保留任务相关信息才能完成重建

训练完成后，VLA 和 RL token 模块全部冻结。

### 第二阶段：在线 RL 精化动作块

在冻结的 RL token 之上，训练**轻量级 actor-critic**（2-3 层 MLP，仅数百参数）：

1. **以 VLA 参考动作为条件输入**：Actor 的输入不仅包括 RL token + 本体感知状态，还包括 VLA 采样的参考动作块 $\tilde{\mathbf{a}}_{1:C}$。这让 actor 从"精化"而非"从零生成"起步。

2. **BC 正则化锚定**：Actor 的损失函数在最大化 Q 值的同时，额外加入对参考动作的 L2 正则化项 $\beta \|\mathbf{a} - \tilde{\mathbf{a}}\|^2$，防止策略偏离 VLA 先验过远。

3. **参考动作 Dropout**：训练时以 50% 概率将参考动作置零，迫使 actor 必须独立学习动作生成能力，避免机械复制 VLA 输出。

4. **动作分块（C=10）**：RL 在长度为 C 的动作块上操作（而非单步），缩短了稀疏奖励下时序差分学习的有效视界，使信用分配更高效。

5. **离策略训练 + 高更新比**：使用 TD3 风格的离策略 actor-critic，回放缓冲区汇聚 VLA rollout、RL 自主执行和人类干预数据，更新-数据比高达 5 倍。

### 关键设计直觉

| 设计选择 | 解决的问题 |
|---------|-----------|
| RL token（信息瓶颈） | 压缩高维 VLA 表示为紧凑状态，使轻量 RL 可行 |
| 参考动作条件化 | 将 RL 限制为"局部编辑"VLA 已有行为，而非从零搜索 |
| BC 正则化 | 防止策略崩溃，保持探索在 VLA 先验附近 |
| 动作分块（C=10） | 缩短 TD 学习视界，使稀疏奖励能有效传播 |
| 参考动作 dropout | 防止 actor 退化为简单复制 VLA 输出 |

本质上，RLT 将 VLA 视为"感知主干 + 行为先验提供者"，在其之上叠加一个快速适应的 RL 层，实现了**大模型泛化能力与小模型样本效率的结合**。

## Q2: Reward 的赋予是以 action chunk 为单位还是 episode 为单位？

**以 episode（或关键阶段结束）为单位，稀疏二值奖励。**

- 整个 episode 只在最后一步收到一个二值奖励：$r_T = 1$（成功），$r_T = 0$（失败）
- 中间所有步的奖励均为 0
- Chunk 级别的 Q 值公式为：

$$Q^{\pi}(\mathbf{s}_t, \mathbf{a}_{t:t+C-1}) = \sum_{t'=t}^{t+C-1} \gamma^{t'-t} r_{t'} + \gamma^{C} \mathbb{E}_{\mathbf{a}'}\left[ Q^{\pi}(\mathbf{s}_{t+C}, \mathbf{a}') \right]$$

由于 $r_{t'}$ 在中间步全为 0，chunk 内的奖励求和项几乎总是 0，真正有价值的信号通过 $\gamma^{C}$ 折扣后的下一 chunk 的 Q 值逐步反向传播。

**为什么分块对稀疏奖励很重要**：
如果不用 action chunk（$C=1$），在 $50\mathrm{Hz}$ 控制频率下，一个持续 5-20 秒的关键阶段对应 **250-1000 步**的信用分配链。单步 RL 方法（HIL-SERL、PLD）在这种超长视界稀疏奖励下几乎无法有效学习。Chunk 将有效视界缩短了 $C$ 倍。

## Q3: 训练时以 VLA 的 token 为输入，为什么还要加一个编码器和解码器？

VLA 的原始 token 嵌入不适合直接作为 RL 的状态输入，编码器-解码器解决了三个具体问题：

### 1. 降维压缩

VLA（$\pi_{0.6}$）的输出是一组 token 嵌入序列 $\mathbf{z}_{1:M}$，$M$ 是 token 数量（对应多张图片、语言指令等），每个 token 是高维向量。直接将其展平作为 RL 状态会导致：
- 状态空间维度过高，轻量级 actor-critic（2-3 层 MLP）难以有效学习
- 样本效率极差——高维输入 + 稀疏奖励 + 少量真实数据 = 学不动

编码器-解码器的信息瓶颈将整个序列压缩为**一个**紧凑向量 $\mathbf{z}_{\mathrm{rl}}$，维度大幅降低。

### 2. 任务相关性筛选

VLA 的 token 嵌入包含**通用知识**（来自大规模预训练），但并非所有信息都与当前任务相关。编码器在**任务特定演示数据**上通过重建损失训练，只有对重建 VLA 嵌入必要的任务相关信息才会被保留在 RL token 中，无关信息被自然过滤。

**证据**：消融实验中，把 RL token 换成 ImageNet 预训练的 ResNet-10 编码器后，吞吐量直接下降 50%——ResNet 的特征是通用视觉特征，缺乏当前操作任务的结构化信息。

### 3. 固定长度接口

VLA 的 token 序列长度 $M$ 随输入变化（不同任务/不同相机数量），而 RL 算法需要一个固定维度的状态向量。编码器-解码器将任意长度的序列统一映射到固定长度的 RL token，提供了稳定的策略输入接口。

**类比**：VLA 是一本厚重的百科全书（通用知识），RL token 是从中提取的一张"任务便签"——只记录当前任务需要的关键信息，格式紧凑，RL 策略拿着这张便签就能快速决策。

## Q4: 关键阶段的针对性改进在算法上是如何实现的？

### 训练时：人类手动切换 + 选择性存储

这不是靠奖励函数自动识别的，而是通过**硬切换 + 选择性存储**实现的：

```
每个 episode 的执行流程：

1. t=0 开始 → 执行基础 VLA 策略（冻结）
   - 这些步的转移数据 不存入 回放缓冲区 B
   - 这个阶段不涉及任何 RL 学习

2. 人类操作者按下"切换" → 控制权移交 RL 策略
   - 当前状态 s 的 RL token z_rl 被提取
   - Actor 开始输出动作块 a_{1:C} ~ π_θ(·|x, ã)
   - 从此刻起的转移数据 存入 B

3. 关键阶段结束 → 人类操作者给出终端信号
   - 成功: r = 1
   - 失败: r = 0
   - 这个终端奖励标记在关键阶段最后一个转移上
```

### 各阶段的数据流

| 阶段 | 谁控制 | 数据是否存 B | 奖励 |
|------|--------|-------------|------|
| 非关键阶段 | 基础 VLA | **否** | 无 |
| 关键阶段（中间步） | RL actor | **是** | 全 0 |
| 关键阶段（终点） | RL actor | **是** | 成功=1 / 失败=0 |

本质上，**关键阶段被隔离成一个独立的子 MDP**：初始状态由 VLA 执行结果决定（带自然变化），稀疏奖励在关键阶段终点给出，只有关键阶段的数据参与 RL 训练。

## Q5: 部署时模型如何判断何时应用 RL 策略？

训练结束后做一次**短微调**：在 VLA 上附加一个二分类头，训练目标是预测"当前时刻是否应该切换到 RL 策略"。监督标签来自训练期间人类操作者的切换时机。

### 数据集构建

将 VLA 在线 RL 训练过程中产生的数据拿出来，复用为标准的行为克隆数据：

| 组件 | 内容 |
|------|------|
| **输入（观测）** | 当前时刻的多模态观测：多张相机图像 + 语言指令 + 本体感知状态 |
| **标签（动作）** | 二值决策——人类操作者在此刻是否按下了切换 |

即标准的**观测-动作对**：VLA 学习的是**状态条件决策**——当螺丝刀头接近螺丝、接头对准端口等**视觉-语义条件**满足时输出切换信号，与绝对时间无关。

```
一个 episode 的时间线:

|──── 基础 VLA 执行 ────|──── RL 关键阶段 ────|
 步1  步2  步3  ...  步k  步k+1  步k+2  ...  步n
  ↓    ↓    ↓         ↓
标签0 标签0 标签0    标签1    (RL阶段不纳入二分类训练)
                      ↑
                 人类在此按下切换
```

### 部署/测试时

```
每个时间步：
    VLA 推理 → 同时输出:
        (1) 参考动作块 ã（正常输出）
        (2) 切换信号：是/否

    如果切换信号 = 否:
        直接执行 VLA 动作
    如果切换信号 = 是:
        提取 RL token → Actor 输出精化动作 → 执行
```

### 为什么可行

关键阶段通常有**明显的视觉-语义边界**：

| 任务 | 切换时机 | VLA 能识别 |
|------|---------|-----------|
| 螺丝安装 | 螺丝刀已对准、即将接触螺丝头 | 距离/姿态接近目标 |
| 扎带紧固 | 双臂已握持扎带两端、准备插入 | 扎带在视野中且位置就绪 |
| 以太网插入 | 接头已移至端口前方 | 端口与接头接近对齐 |
| 充电器插入 | 充电器已对准插座 | 插座在腕部相机视野中 |

这些场景在 VLA 的嵌入空间中本身就具有区分度，学一个二分类不需要额外收集数据。

这个微调本质上是**一次小规模的 SFT/行为克隆**：用人类操作者在训练期间做出的切换决策作为专家标签，让 VLA 模仿"何时切换"这个行为。和 VLA 本身学习"何时执行什么动作"是同样的范式，只是这次学的动作是二值的"切换/不切换"而非连续的机械臂控制量。
